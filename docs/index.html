关键洞察

CANN testing-infra 问题最为集中：60 个 testing-infra issue 占 CANN 基础设施类的 27.9%（60/215），远超其他子分类。主要涉及 UT 覆盖率提升、ATK 测试用例补齐、测试框架 opbuild 适配、代码覆盖率工具迁移等系统性问题，反映出 CANN 组织在测试基础设施方面的大规模补课需求。
CANN 与 Ascend 基础设施规模相近但结构不同：CANN 215 个（25.1%）vs Ascend 79 个（19.9%），占比接近但分布差异显著。CANN 以 testing-infra（60）、build（33）、code-quality（29）为主；Ascend 则以 ci/cd（18）、testing-infra（16）、build（12）为主。CANN 的 code-quality 和 dev-environment 关注度明显高于 Ascend。
Ascend/pytorch CI 稳定性是核心痛点：12 个基础设施 issue 中 ci/cd 占 6 个（50%），远超其他仓库的 ci/cd 占比。问题集中在流水线编译报错、CI 环境 ACL 头文件缺失、门禁 x86 覆盖不足、workflow 定时策略调整等方面。
BoostKit 基础设施关注度为零：19 个 issue 覆盖 8 个仓库，全部为功能需求、CVE 安全漏洞和文档反馈，无任何 CI/测试/构建类 issue。规模虽小但基础设施类完全缺失，可能问题未被有效识别或报告渠道不畅。
testing-infra 是全社区第一大基础设施类别：合计 79 个（占 25.8%），在 CANN、Ascend 两个主要社区中均位列前三。测试用例同步移植、UT 覆盖率、ATK 测试框架、代码覆盖率工具等是跨社区共性主题。
