关键洞察

openGauss 首次纳入，基础设施占比最低（11.1%）：63 个 issue 中仅 7 个基础设施类，以构建系统（3）和开发体验（2）为主。该社区以数据库产品为核心，issue 主要为 SQL 功能 bug、文档翻译和 CVE 漏洞，基础设施关注度显著低于其他社区。
openEuler 工具链类异军突起（46 个）：本轮分类中 toolchain 占 openEuler 基础设施的 31.5%，主要来自 ag-cli、skills（RAG 工具）和 ubs-comm 等开发者 CLI 工具仓库。与 CANN 的 testing-infra（57）、Ascend 的 testing-infra（22）形成三种不同的基础设施格局。
CANN 保持最大规模且分类最多样：856 个 issue、236 个基础设施类、13 个子分类均有覆盖。其中 other-infra（21）和 deployment（6）的出现反映出该社区基础设施需求的持续扩展，不再局限于传统的测试-构建-工具链三角。
BoostKit 连续三个周期零基础设施关注：累计 19 个 issue 无一基础设施类，是 6 个社区中唯一的零值。建议社区建立 CI/测试基础设施问题识别与上报机制。
testing-infra 是全社区第一大类别（103 个）：在 CANN（57）、Ascend（22）、openEuler（21）中均位列前列，UT 覆盖率、测试框架适配、Golden 数据管理是跨社区共性主题。toolchain（71）因 openEuler 驱动跃居第二，ci/cd（69）和 build（70）紧随其后。
