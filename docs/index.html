关键洞察

CI/CD 是 Ascend/pytorch 最突出问题：12 个基础设施 issue 中 ci/cd 占 6 个（50%），涉及流水线编译报错、CI 环境 ACL 头文件缺失、门禁 x86 覆盖不足、workflow 定时策略调整等。CI 稳定性是该仓库最紧迫的基础设施改进方向。
CANN 构建系统问题高度集中：74 个基础设施 issue 中 build 类占 30 个（40.5%），远超其他分类。主要涉及 CMake 配置、编译依赖管理、UT 编译失败、源码编译包输出等。该组织在构建系统的标准化和自动化方面有较大改进空间。
Ascend 组织基础设施需求多样化：79 个 issue 分布在 11 个子分类中，覆盖 25 个仓库。除 ci/cd（18）和 testing-infra（16）外，developer-experience（11）和 monitoring（8）也占较高比例，反映出组织级工具链的复杂性和多维度改进需求。
BoostKit 缺乏基础设施投入关注：19 个 issue 全部为功能需求、CVE 漏洞和文档反馈，无任何基础设施类问题。该社区规模虽小，但完全缺乏 CI/测试/构建类 issue 可能意味着基础设施问题未被有效识别或报告。
测试基础设施是跨社区共性痛点：testing-infra 全社区合计 29 个（占基础设施类 17.6%），在 CANN、Ascend、Ascend/pytorch 三个社区中均位列前三。测试用例同步移植、NPU 适配 patch 覆盖不足、门禁用例拆分等是高频主题。
